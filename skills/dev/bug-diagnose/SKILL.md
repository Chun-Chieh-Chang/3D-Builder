# Bug Diagnose (結構化錯誤診斷)

此技能用於對抗 Vibe Coding 中的「盲目修復 (Guess & Check)」行為。
當用戶回報 Bug 或系統出現非預期錯誤時，你必須停止一切編碼動作，嚴格依照以下四階段診斷流程執行。

## 診斷流程 (The Diagnostic Workflow)

### 1. 🔍 Reproduce (重現問題)
- **目標**：確認問題確實存在，並找出觸發條件。
- **行動**：
  - 詢問用戶重現步驟 (Steps to reproduce)。
  - 釐清環境狀態（OS、瀏覽器、依賴版本等）。
  - 若有錯誤日誌 (Log) 或堆疊追蹤 (Stack Trace)，務必詳細閱讀。

### 2. ✂️ Minimize (最小化影響範圍)
- **目標**：剝離不相關的代碼，定位最核心的脆弱點。
- **行動**：
  - 透過二分法或隔離測試，確認是哪一段具體的邏輯、模組或狀態流出錯。
  - 將問題限縮到幾行代碼或單一 API 內。

### 3. 🧠 Hypothesise (提出假設與根因分析 RCA)
- **目標**：基於事實，提出符合邏輯的錯誤原因假設。
- **行動**：
  - 在開始修改前，將你的假設寫入 `DEV_LOG.md` 的 RCA (Root Cause Analysis) 區塊中。
  - 向用戶解釋：「為什麼會發生這個錯誤？」
  - 提出修復策略，並分析該策略是否會產生副作用 (Regression)。

### 4. 🛠️ Fix & Verify (精準修復與 CAPA)
- **目標**：以最小的代碼變動 (Surgical Change) 修復問題，並驗證結果。
- **行動**：
  - 執行修復代碼。
  - 在 `DEV_LOG.md` 記錄 CAPA (矯正與預防措施)，說明未來如何避免此類錯誤。
  - 要求用戶或測試工具確認錯誤已消除且無其他副作用。

## ⚠️ 絕對禁忌
- **禁止瞎猜 (No Guessing)**：沒有確認根因前，絕對不准給出「試試看這樣改」的代碼。
- **禁止連帶破壞**：修復時嚴禁修改與問題無關的正常代碼（保持 Diff 的絕對乾淨）。
