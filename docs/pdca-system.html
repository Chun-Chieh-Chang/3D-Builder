👤 使用者指令 / SOP 輸入 INPUT

輸入建模目標（例如 YouTube 影片網址或 SolidWorks 練習題 SOP）。系統會自動提取字幕、文字規格或工程參數。

🧠 SolidWorks 專家代理 PLAN

負責將繁雜的建模需求拆解為精確的 CAD 特徵樹操作序列（例如：基準面選擇、草圖繪製、幾何約束約束、拉伸/旋轉特徵建立），提供明確的指導步驟。

🤖 UI 實作機器人 (Robot) DO

系統核心樞紐 (Pivot)。接手專家給予的步驟，實際控制瀏覽器子代理，在網頁端 UI 執行滑鼠點擊、參數輸入與草圖繪製，以對齊 SolidWorks 真實建模路徑。

📐 架構師代理 (Architect) ANALYZE

當實作機器人卡關（如找不到按鈕、特徵不支援、發生JS錯誤）時觸發。架構師介入審查錯誤日誌與代碼庫，定位最脆弱點，提供精準修補方案。

🔪 核心實作代理 (Implementer) ACT/DO

負責實施架構師的修復提案，進行外科手術式的代碼修改（Surgical Edits），不帶引入副作用的前提下，精準修復代碼脆弱環節。

🛡️ 品質保證代理 (QA) CHECK

在代碼修改完畢後，執行回歸掃描、單元測試與 build 流程完整性驗證。確保專案在零紅色警告狀態下，自動將控制權移交回實作機器人重試。

🎉 任務完成 (Success Output) DONE

實作機器人在不遭遇阻礙的情況下順暢跑通所有建模步驟，3D模型正確生成並與後端 Mass Properties 驗證完全一致，輸出最終特徵模型。

💡 設計亮點：
1. 零交叉 (Cross-Free) 布局： 將「任務完成！」移至中央主軸，騰出左側的完全空白通道供橘色的「反饋重試」線條流經，彻底解決了舊版線條橫穿 green 卡片產生的視覺干擾。
2. 高對比色彩層次： 嚴格遵照色彩大師規範，使用微調 Slate/Cool Gray 作為背景，提供清透的卡片懸浮投影，標示明顯的 Phase 進度。
3. 自動修復自愈閉環： Robot (DO) 遇到 Blocker 時向右切換至修復流程，通過 QA 驗證後，藉由反饋線迴路重新注入 Robot，實現真正無人值守的自愈工程。
